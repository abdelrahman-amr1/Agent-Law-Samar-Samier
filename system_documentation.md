# دليل شرح النظام التقني (سَنَد | شريكك القانوني الذكي)

هذا الملف يمثل دليلاً تقنياً شاملاً لهندسة وآلية عمل منصة "سَنَد | شريكك القانوني الذكي" (سَنَد). صُمم هذا الدليل لمساعدتك في شرح تفاصيل المشروع لأي مبرمج جديد سينضم للعمل على التطبيق.

---

## 1. نظرة عامة على المشروع (Project Overview)
التطبيق عبارة عن منصة (SaaS) مخصصة للمحامين، تعمل كمساعد قانوني ذكي يعتمد على الذكاء الاصطناعي التوليدي (Gemini API). 
يقوم النظام بتحليل وقائع القضايا (نصوص أو ملفات مرفقة)، والبحث في مراجع قانونية سحابية، ثم تقديم استشارات، أو صياغة مذكرات دفاع، أو كتابة صحف دعاوى جاهزة للطباعة بالصيغة القانونية المصرية الصحيحة.

## 2. التقنيات المستخدمة (Tech Stack)
* **واجهة المستخدم (Frontend):** React.js باستخدام إطار عمل **Next.js** (App Router).
* **لغة البرمجة:** TypeScript / JavaScript.
* **قاعدة البيانات والمصادقة (Backend as a Service):** **Supabase** (PostgreSQL).
* **الذكاء الاصطناعي (AI):** **Google Gemini API** (نموذج `gemini-3-flash` حالياً).
* **التصميم (Styling):** CSS عادي مخصص مع أيقونات (Lucide-react).

---

## 3. الميزات الأساسية للنظام (Core Features)

### أ. نظام المصادقة وإدارة الحسابات (Authentication & User Management)
* يعتمد النظام على `Supabase Auth` لتسجيل الدخول.
* يوجد جدول `profiles` مرتبط بالمستخدمين يحدد نوع الحساب (Admin أو User) وحالته (`is_active`).
* إذا تم إيقاف حساب المستخدم (`is_active = false`) ولم يكن Admin، يقوم النظام بتسجيل خروجه تلقائياً وتوجيهه لصفحة الدخول برسالة تفيد بإيقاف حسابه.

### ب. إدارة القضايا (Cases Management)
* يمكن للمحامي إنشاء عدة "قضايا". كل قضية لها ID خاص بها وتُحفظ في جدول `cases`.
* عند فتح المحادثة، يقوم النظام بجلب رسائل هذه القضية فقط من جدول `messages`.

### ج. محرك الذكاء الاصطناعي (AI Engine & Context)
* **السياق (Chat History):** عند إرسال طلب جديد، يقوم النظام بجمع تاريخ المحادثة بالكامل (Chat History) لتلك القضية وإرساله للـ API لكي يتذكر المساعد سياق الحديث (مثلاً: طلب المستخدم استشارة ولا يريد كتابة دعوى).
* **الموجه النظامي (System Prompt):** يمتلك المساعد أوامر صارمة (`src/lib/systemPrompt.ts`) تحتم عليه:
  1. سؤال المحامي عن نوع الخدمة (استشارة، دعوى، مرافعة).
  2. أخذ تفاصيل القضية دون إزعاج.
  3. رفض التحدث في تفاصيل أي قضية أخرى غير المفتوحة.
  4. تنسيق الردود بشكل قانوني جاهز للطباعة.

### د. المراجع القانونية السحابية والمحلية (Law Files & RAG approach)
لكي يكون المساعد دقيقاً في القانون المصري، نقوم بتغذيته بملفات مراجع:
* **الملفات المحلية:** الموجودة في مجلد `file/`.
* **الملفات السحابية:** المحفوظة في `Supabase Storage` (في Bucket يسمى `law_files`).
* يقوم النظام بجلب هذه الملفات ورفعها إلى سيرفرات Gemini، ثم يرفقها مع كل سؤال يسأله المستخدم.

### هـ. التخزين المؤقت للملفات (Gemini Cache)
* نظراً لأن رفع الملفات القانونية الكبيرة (PDFs) إلى Gemini في كل رسالة سيستهلك وقتاً طويلاً جداً، تم إنشاء جدول `gemini_cache` في Supabase.
* يقوم النظام برفع الملف مرة واحدة للـ Gemini API ويحصل على `File URI`. 
* يتم حفظ هذا الـ URI في قاعدة البيانات ومدة صلاحيته 48 ساعة. إذا قام المستخدم بسؤال جديد خلال 48 ساعة، يستخدم النظام الـ URI المخزن مسبقاً بدلاً من إعادة رفع الملف، مما يجعل الرد فائق السرعة.

### و. طباعة المذكرات (PDF Export)
* بعد أن يكتب المساعد مذكرة الدفاع أو الدعوى، يوجد زر لطباعة الرد (PDF).
* يقوم الكود بفتح نافذة جديدة (Window) بتنسيق CSS خاص وجاهز للطباعة بحجم ورقة A4، وخطوط رسمية (Amiri)، ويستدعي أمر `window.print()` لطباعة المذكرة فوراً.

## 4. هيكل مجلدات الكود (Project Structure)

* `src/app/page.tsx`: الواجهة الرئيسية (لوحة المحامي)، تحتوي على قائمة القضايا، الدردشة، ورفع الملفات، والطباعة.
* `src/app/login/page.tsx`: صفحة تسجيل الدخول والتعامل مع Supabase Auth.
* `src/app/api/chat/route.ts`: الـ Backend API الذي يستقبل رسالة المحامي، يجلب المراجع القانونية، يتأكد من الـ Cache، ويرسل الطلب بالكامل لـ Gemini ليعود بالرد.
* `src/lib/geminiApi.ts`: ملف الدوال المسؤولة عن التخاطب المباشر مع سيرفرات جوجل (رفع الملفات، وإرسال الـ Prompt واستقبال الرد).
* `src/lib/systemPrompt.ts`: التعليمات البرمجية التي تبرمج عقل الذكاء الاصطناعي وتقول له كيف يتصرف (شخصية المحامي).
* `src/lib/supabase.ts`: إعدادات الاتصال بقاعدة بيانات Supabase.
* `supabase.sql`: ملف يحتوي على هيكل الجداول الأساسية لقاعدة البيانات (لمساعدة المبرمج في إنشاء الجداول إذا لزم الأمر).

---

## 5. ملاحظات هامة للمبرمج (Important Notes for the Developer)
1. **Gemini API Key:** يتم حفظه في جدول `settings` في قاعدة البيانات. ولا يعمل الذكاء الاصطناعي إلا إذا أضاف الأدمن الـ API Key الخاص به. 
2. **نوع النموذج (Model):** الكود يستخدم حالياً نموذج `gemini-3-flash` لأن حساب العميل مجاني/محدود وهذا النموذج يوفر حصة كافية للاستخدام وتجاوز الضغط. يمكن تغييره بسهولة في ملف `geminiApi.ts`.
3. **مكتبات خارجية غائبة:** النظام لا يعتمد على مكتبات ضخمة معقدة لتسهيل صيانته، الـ UI مبني بـ CSS و React Hooks العادية لتجربة سريعة وخفيفة.
